## Root cause

There are two compounding issues:

**1. CIB variants were seeded with the Loose price.** When a new retro game
product is created in this store, both variants start out at the same price.
Subsequent price-refresh jobs are supposed to update each variant independently
based on PriceCharting's loose-vs-CIB market data.

**2. PriceCharting search has ~22% miss rate on long-tail titles.** The last
`search-price-refresh.py` run on 2026-04-08 only matched 2,736 of 6,121
products (the rest were either no_match or didn't have CIB market data on
PriceCharting). The unmatched products kept their seeded "CIB == Loose"
pricing forever.

`full-price-refresh.py` (which scrapes per-console catalog pages instead of
search) has its own matching bug — the last run on 2026-04-06 only matched
355 out of 6,121 products (5.8%) because its exact-title normalization is
brittle against Shopify's title formats.

## False alarms investigated and dismissed

- **"2,500 CIB variants updated today"**: All CIB variants showed `updatedAt`
  in 2026-04-11 in the scan. This is an artifact of `fix-cib-inventory.py`
  which ran earlier today and set `inventoryPolicy = CONTINUE` on every CIB
  variant — Shopify ticks `updatedAt` on any field change, not just price.
- **"PriceCharting search is broken / games_table no longer exists"**: False.
  The search URL 301-redirects from `type=videogames` to `type=prices` but
  the resulting HTML still has the `#games_table` selector with the same
  column layout. `search-price-refresh.py` works fine, as verified by a
  direct call to `search_pricecharting("Mystical Ninja Starring Goemon",
  "Nintendo 64")` which returned the correct loose / CIB / new prices.

## Fix

**Script:** `scripts/fix-cib-equals-loose.py`

Targeted fix that only touches the 1,364 products where CIB == Loose. For
each product it:

1. Searches PriceCharting for `{clean_title} {console}`
2. If a hit comes back with `cib_market > 0`, recalculates both variant
   prices at 1.35x market and updates both.
3. Falls back to a **1.8x loose uplift** when PriceCharting has no CIB data —
   the retro CIB premium is typically 1.5x–3x so this is a safe floor that
   unsticks the pricing without overshooting.
4. Enforces a **minimum $5 premium** on CIB over loose as a belt-and-suspenders
   sanity check.
5. Uses the same "round DOWN to $X.99" psychology-price rounding as
   `manage-sales.py`.

Diff threshold: only actually writes an update when the change is >= $1.00
in either direction, so trivial churn is filtered out.

Invocation:
```bash
# Dry run
python3 scripts/fix-cib-equals-loose.py

# Apply
python3 scripts/fix-cib-equals-loose.py --apply

# Skip PriceCharting search (faster — just apply 1.8x uplift fallback)
python3 scripts/fix-cib-equals-loose.py --apply --uplift-only
```

Input: `data/cib-equals-loose.json` (generated by the fresh 2026-04-11
catalog scan — regenerate it before re-running).

Output: `data/logs/fix-cib-equals-loose-{timestamp}.{csv,log}`
